# Política de seguridad

## Versiones soportadas

La rama `main` representa la única versión mantenida y desplegada en producción.

## Reporte de vulnerabilidades

No publiques vulnerabilidades, credenciales ni datos personales en un issue público.

Envía el reporte a `scriptordigitaloficial@gmail.com` con:

- Una descripción clara del problema.
- La ruta o componente afectado.
- Pasos mínimos para reproducirlo.
- Impacto estimado.
- Cualquier evidencia que no contenga datos de terceros.

El reporte será revisado de forma privada. Una corrección y su divulgación pública se coordinarán después de reducir el riesgo para los usuarios.

## Secretos

Las claves de Resend, WhatsApp y cualquier proveedor externo deben configurarse como variables de entorno. Nunca deben incorporarse al repositorio, capturas, issues o registros públicos.
